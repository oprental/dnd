# Хроники подземелья — деплой

Этот сайт состоит из трёх частей:

- `index.html` — сама игра (фронтенд)
- `api/chat.js` — серверная функция, которая обращается к Grok (xAI) с твоим ключом (ключ хранится только на сервере)
- `api/room.js` — серверная функция для хранения комнат/анкет игроков (использует Upstash Redis, бесплатно)

## 1. Заведи бесплатную базу на Upstash

1. Зайди на https://console.upstash.com и зарегистрируйся (можно через GitHub).
2. Создай новую **Redis** базу (Create Database), регион — любой ближайший.
3. Открой созданную базу → вкладка **REST API**.
4. Скопируй значения `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN` — они понадобятся на шаге 3.

## 2. Задеплой на Vercel

Проще всего — через GitHub:

1. Создай новый репозиторий на GitHub и залей туда все файлы из этой папки (`index.html`, `api/`, `package.json`).
2. Зайди на https://vercel.com, зарегистрируйся (можно через GitHub) и нажми **Add New → Project**.
3. Выбери свой репозиторий и нажми **Deploy** (настройки по умолчанию подходят — Vercel сам увидит папку `api/` и превратит файлы в серверные функции).

Либо через терминал, если знаком с CLI:

```bash
npm i -g vercel
cd dnd-site
vercel
```

## 3. Добавь секретные ключи

В панели Vercel: **Project → Settings → Environment Variables** — добавь три переменные (для окружения Production и Preview):

| Имя | Значение |
|---|---|
| `GROK_API_KEY` | твой ключ с https://console.x.ai |
| `UPSTASH_REDIS_REST_URL` | из шага 1 |
| `UPSTASH_REDIS_REST_TOKEN` | из шага 1 |

После добавления переменных сделай **Redeploy** (Deployments → ⋯ → Redeploy), чтобы они применились.

## 4. Готово

Открой выданный Vercel адрес (вида `https://твой-проект.vercel.app`) — это и есть публичный сайт. Можно открывать с разных устройств, комнаты и анкеты теперь синхронизируются через Upstash, а истории генерирует Grok через `/api/chat`.

## Локальный запуск (по желанию)

```bash
npm i -g vercel
cd dnd-site
vercel dev
```

Это поднимет сайт на `localhost` вместе с серверными функциями — удобно для отладки перед деплоем. Не забудь создать локальный файл `.env.local` на основе `.env.example` с теми же тремя ключами.

## Если что-то не работает

- Ошибка про `GROK_API_KEY` — переменная не задана или не применилась (нужен Redeploy после добавления).
- Комнаты не находятся — проверь `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN`.
- Сцена не рисуется — это нормально в единичных случаях, Grok иногда возвращает невалидный SVG, тогда подставляется запасная простая картинка.
